1. システム概要
================================================================================

本システムは、テーマ（例: Linux、Docker、Python）を入力すると、
Groq API（無料枠あり）上の AI が技術まとめ記事を自動作成し、
JSON ファイルとして保存して Web サイトに公開するツールです。

【最終目標】
  テーマ入力 → AI記事生成 → 保存 → 自動サイト化
  を一連の流れで行い、将来的には複数ジャンルのサイトを量産できる構成。

【特徴】
  - React 不使用（HTML / CSS / Vanilla JS のみで軽量）
  - 記事は /articles/*.json に保存（データベース不要）
  - サーバー側で HTML を生成（SEO に有利）
  - 拡張用の自動化フックを用意

2. 何ができるか（機能一覧）
================================================================================

■ 記事生成（管理画面 /admin）
  - テーマ入力（例: 「Linuxの基本コマンド」）
  - 切り口の指定（任意。例: 「初心者向け」）
  - クイック選択チップ（Linux, Git, Docker など）
  - AI による自動生成:
      ・タイトル
      ・概要（summary）
      ・本文（HTML、h2/h3 見出し付き）
      ・まとめ
      ・タグ（複数）
      ・カテゴリ
      ・slug（URL用英字）
      ・SEO用 meta title / description
  - プレビュー表示
  - 下書き保存 / 公開して保存

■ 記事管理（管理画面）
  - 記事一覧表示
  - 公開 ↔ 下書き の切り替え
  - 記事削除
  - 公開記事のプレビューリンク

■ 公開サイト（一般ユーザー向け）
  - ホーム（/）: 公開記事のカード一覧
  - 検索（タイトル・概要・タグから絞り込み）
  - カテゴリフィルタ
  - 記事詳細（/article/:slug）: 全文表示

■ SEO
  - meta title / meta description
  - canonical URL
  - OGP（og:title, og:description など）
  - JSON-LD（BlogPosting 構造化データ）

■ 運用・診断
  - 起動時に Groq API キーの検証
  - 管理画面で API 接続状態の表示
  - start.bat / restart.bat による簡易起動

3. 全体の仕組み（データの流れ）
================================================================================

【記事を1本つくる流れ】

  [管理者]
      │
      ▼
  管理画面 (/admin) でテーマ入力
      │
      ▼
  「AIで記事を生成」ボタン
      │
      ▼
  ブラウザ → POST /api/articles/generate
      │
      ▼
  サーバー (Express) → utils/groq.js
      │
      ▼
  Groq API (llama-3.3-70b-versatile)
      │  ・システムプロンプト（執筆ルール）
      │  ・ユーザープロンプト（テーマ＋切り口）
      │
      ▼
  AI が JSON 形式で記事データを返す
      │
      ▼
  サーバーが JSON をパース・検証
      │
      ▼
  管理画面にプレビュー表示
      │
      ▼
  「公開して保存」または「下書き保存」
      │
      ▼
  ブラウザ → POST /api/articles
      │
      ▼
  utils/articleStore.js
      │  ・slug の重複チェック
      │  ・SEO メタ情報の付与
      │  ・HTML のサニタイズ
      │
      ▼
  /articles/{slug}.json に保存
      │
      ▼
  ホーム (/) または記事ページ (/article/:slug) で表示


【サイト表示の流れ】

  訪問者が URL にアクセス
      │
      ▼
  routes/pages.js がリクエストを受信
      │
      ▼
  articles/*.json から記事データを読み込み
      │
      ▼
  templates/*.html + utils/render.js で HTML を組み立て
      │  （SEO タグ・本文・OGP を埋め込み）
      │
      ▼
  完成した HTML をブラウザに返す

6. 記事生成の仕組み（AI）
================================================================================

【使用API】
  Groq API（OpenAI 互換）
  ベースURL: https://api.groq.com/openai/v1
  デフォルトモデル: llama-3.3-70b-versatile

【プロンプトの役割】（utils/prompt.js）
  - 初心者向け・具体例あり・専門用語に説明
  - テーマごとに内容を変える（テンプレ使い回し禁止）
  - 出力は JSON のみ（title, summary, body, conclusion, tags, category, slug 等）

【AIが返すデータ例（概念）】
  {
    "title": "Linux基本コマンド入門",
    "summary": "初心者が覚えるべきコマンドを解説…",
    "body": "<h2>はじめに</h2><p>…</p>…",
    "conclusion": "まとめの文章…",
    "tags": ["Linux", "初心者", "CLI"],
    "category": "Linux",
    "slug": "linux-basic-commands",
    "metaTitle": "…",
    "metaDescription": "…"
  }

【保存時の追加処理】
  - slug が重複する場合は末尾に -1, -2 を付与
  - SEO 情報を utils/seo.js で再計算・マージ
  - body から <script> 等を除去（sanitize.js）
  - status を draft（下書き）または published（公開）に設定

7. 記事データの形式
================================================================================

保存先: articles/{slug}.json

【主なフィールド】

  フィールド          説明
  ─────────────────────────────────────────
  id                  内部ID（タイムスタンプベース）
  slug                URL用（例: linux-basic-commands）
  title               記事タイトル
  summary             概要（一覧・SEO用）
  body                本文（HTML）
  conclusion          まとめ文
  tags                タグ配列
  category            カテゴリ名
  theme               生成時に入力したテーマ
  status              "draft" または "published"
  createdAt           作成日時（ISO8601）
  updatedAt           更新日時
  publishedAt         公開日時（公開時のみ）
  metaTitle           SEOタイトル
  metaDescription     SEO説明文
  ogTitle / ogDescription  OGP用
  canonical           正規URL
  jsonLd              構造化データ（JSON-LD）

8. SEOの仕組み
================================================================================

記事保存時・表示時に utils/seo.js が以下を自動設定します。

  ・<title> … metaTitle（なければ「タイトル | サイト名」）
  ・<meta name="description"> … 検索結果用の説明文
  ・<link rel="canonical"> … 正規URL（重複コンテンツ防止）
  ・OGP … og:title, og:description, og:url, og:type
  ・Twitter Card … summary_large_image 形式
  ・<script type="application/ld+json"> … BlogPosting スキーマ

ホームページは WebSite スキーマを使用。

サイト名・ベースURLは .env の SITE_NAME, SITE_URL から取得。

9. API一覧
================================================================================

【診断】
  GET  /api/health
       … サーバー稼働確認
       ?verify=true … Groq APIキーの実効性も検証

【記事】
  GET    /api/articles
         … 一覧（クエリ: q=検索, category=, tag=, all=true で下書き含む）

  GET    /api/articles/:slug
         … 1件取得

  POST   /api/articles/generate
         … AI生成（body: { "theme": "Linux", "angle": "任意" }）

  POST   /api/articles
         … 保存（body: { "draft": {...}, "theme": "...", "publish": true/false }）

  PATCH  /api/articles/:slug
         … 更新（例: { "status": "published" }）

  DELETE /api/articles/:slug
         … 削除

10. 使い方（操作手順）
================================================================================

【初回セットアップ】
  1. Node.js LTS をインストール
  2. プロジェクトフォルダで npm install
  3. .env.example をコピーして .env を作成
  4. Groq Console で API キー取得 → GROQ_API_KEY に全文貼り付け
  5. restart.bat をダブルクリック
  6. 黒い画面に「Groq API → OK」と表示されることを確認

【記事を1本つくる】
  1. ブラウザで http://localhost:3000/admin を開く
  2. テーマを入力（またはチップをクリック）
  3. 「AIで記事を生成」をクリック（数十秒かかることがある）
  4. プレビューを確認
  5. 「公開して保存」→ ホームに表示される
     「下書き保存」→ 管理画面のみ（一般公開されない）

【記事を確認・管理】
  - 管理画面右側の一覧で公開/下書き切替・削除
  - 公開済みは「表示」リンクで /article/:slug を開ける
  - ホーム (/) で検索・カテゴリフィルタ

【.env を変更したとき】
  必ず restart.bat でサーバーを再起動すること。
  再起動しないと古い API キーが使われ 401 エラーになる。

11. 環境設定（.env）
================================================================================

  変数名              説明
  ─────────────────────────────────────────────────
  GROQ_API_KEY        Groq APIキー（gsk_ で始まる全文・50文字程度）
  GROQ_MODEL          モデル名（省略時: llama-3.3-70b-versatile）
  SITE_NAME           サイト名
  SITE_URL            公開URL（例: http://localhost:3000）
  SITE_DESCRIPTION    サイト全体の説明
  PORT                サーバーポート（省略時: 3000）

12. 起動・停止方法
================================================================================

【起動】
  start.bat をダブルクリック
  または: npm start

【再起動（.env 変更後・不調時）】
  restart.bat をダブルクリック

【停止】
  黒いコマンド画面で Ctrl+C

【起動成功の目安】
  Groq API    → OK (llama-3.3-70b-versatile)
  と表示されること

13. トラブルシューティング
================================================================================

  症状                          原因と対処
  ─────────────────────────────────────────────────────────
  localhost が開けない          サーバー未起動 → start.bat 実行
  401 Invalid API Key           ・キーが短い/間違い → .env を修正
                                ・再起動していない → restart.bat
  APIキーが短すぎます           gsk_ から全文コピー（... 不可）
  記事が生成されない            管理画面の赤いエラーを確認
  ホームに記事が出ない          status が published か確認
  Cannot find module 'express' npm install を実行
